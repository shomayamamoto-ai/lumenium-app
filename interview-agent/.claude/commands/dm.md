# /dm — 初回スカウト DM 文面生成

面談前の **「初回コンタクト」** に特化したコマンド。SNS 情報からパーソナライズされた DM を **3 パターン** 生成します。

## 引数
- `$1` : 候補者 slug（`profile.md` のみでも OK、`brief.md` があればなお良し）

## 手順

### Step 1: 情報収集
- `candidates/$1/profile.md` を Read
- `candidates/$1/brief.md` があれば Read（SNS 分析結果を活用）
- brief.md が無い場合は、profile.md の SNS URL から `sns-analyzer` サブエージェントを呼び出して軽量分析

### Step 2: DM の 3 パターン生成
以下の 3 軸で異なる切り口の文面を作る:

**パターン A: 「作品褒め」軸**
- 最新投稿 1 点を具体的に褒める（楽曲名・ダンスルーティン・写真の雰囲気など）
- 褒めて終わるのではなく、スカウトへつなげる
- 長さ: 180–250 字

**パターン B: 「共感・ストーリー」軸**
- 候補者の投稿から読み取れる「志向性」に共感を寄せる
- 自社スクールが提供できる具体的な成長機会 1 点を示す
- 長さ: 200–280 字

**パターン C: 「直球・実績」軸**
- 自社の実績（合格者、契約事例）を先に出す
- 面談オファーを明確に提示
- 長さ: 150–220 字

### Step 3: 各パターンに「選ぶべき条件」を付ける
どのパターンを使うべきかの判断基準を付記する:

```
【パターン A を選ぶべき時】フォロワー多め、承認欲求が読み取れる
【パターン B を選ぶべき時】投稿に「悩み・葛藤」が滲んでいる
【パターン C を選ぶべき時】既にオーディション受験歴など真剣度が高い
```

### Step 4: チェックリスト
各 DM に以下を満たしているかを自己チェックさせる:
- [ ] 候補者の **固有名詞**（投稿内容や楽曲名）が最低 1 つ入っているか
- [ ] **しつこくない** か（一読で負担感を与えない）
- [ ] 未成年の可能性がある場合、保護者同意への言及があるか
- [ ] **次アクションが明確** か（「返信くれたら日程案内」等）
- [ ] コピペ感がない（テンプレ感を削ぐフレーズが含まれているか）

### Step 5: 保存と返却
- `candidates/$1/dm-drafts.md` に Write（3 パターン + 選定ガイド）
- ユーザーには **3 パターンをそのまま表示**し、「パターン B を推奨します（理由: XXX）」と 1 つレコメンド

## 制約
- 絵文字は最大 2 個まで（多すぎると業者感）
- 「〜しませんか?」で終わる疑問形で締める（返信率が上がる傾向）
- 候補者の SNS で実在する内容のみを引用（存在しない投稿を捏造しない）
- 未成年の可能性があれば冒頭に `⚠️ 未成年の可能性あり、保護者同席/同意の旨を必ず文面に含めました` と注意を出す

## 出力例（ユーザーへの最終表示）
```
📨 DM 3 パターン生成: candidates/2026-05-01-tanaka-yuki/dm-drafts.md

━━━ パターン A（作品褒め軸）━━━
{{文面}}

━━━ パターン B（共感軸）━━━
{{文面}}

━━━ パターン C（実績軸）━━━
{{文面}}

🎯 推奨: パターン B
理由: 投稿「周りみんな就職決まっていく中で…」に葛藤が出ており、
      共感軸が最も刺さる可能性が高い。
```
