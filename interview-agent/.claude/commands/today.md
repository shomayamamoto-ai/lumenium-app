# /today — 本日のアクションダッシュボード

候補者フォルダを全部チェックして、**今日対応すべきこと** だけを優先度付きで一覧化します。

## 手順

### Step 1: 全候補者の classification.md を走査
Glob: `candidates/*/classification.md`（`_example` は除外）

### Step 2: 各候補者から以下を抽出
- 氏名（profile.md の `# 候補者プロフィール: XXX` から）
- 分類（本気層 / 迷い層 / 冷やかし）
- フォロー計画テーブルの TODO のうち **期日が今日以前** かつ **状態が「未着手」**
- 次回判定タイミング（`次回判定タイミング: YYYY-MM-DD` の行）

### Step 3: 以下のロジックで優先度付け
| 優先度 | 条件 |
|-------|------|
| 🔴 最優先 | 本気層 かつ 24h TODO 未対応 |
| 🟠 高 | 本気層 かつ 3 日以内 TODO / 迷い層 かつ 24h TODO |
| 🟡 中 | 迷い層 の 7 日・14 日 TODO |
| 🟢 低 | 冷やかし層 の単発 TODO |

### Step 4: /brief, /minutes, /classify が未実行の候補者もチェック
- profile.md はあるが brief.md がない → 「🆕 ブリーフ未生成」
- minutes.md はあるが classification.md がない → 「📝 分類未実行」

### Step 5: Markdown テーブルで一括出力
以下の形式でユーザーに返す:

```
# 📅 {{YYYY-MM-DD}} のアクション

## 🔴 最優先（本気層・今すぐ）
| 候補者 | アクション | 期限 |
|--------|-----------|------|
| 田中由紀 | 契約書 PDF 送付（/followup で文面生成可） | 本日 18:00 |

## 🟠 高優先（本日中）
| ... |

## 🟡 中優先（今週中）
| ... |

## 🟢 定期タスク
| ... |

## 🆕 未処理
- 2026-05-02-sato-miki: ブリーフ未生成（面談日 {{日付}}）
- 2026-04-28-suzuki-ken: 議事録はあるが分類未実行

## 📊 スナップショット
- 総候補者数: N 名
- 分類内訳: 本気層 X / 迷い層 Y / 冷やかし Z / 未分類 W
- 今週の面談数: N 件
```

### Step 6: ワンクリック案内
最後に「今すぐ実行すると良いコマンド」を 1-3 個具体的に提示:

```
次のアクション候補:
  /followup 2026-05-01-tanaka-yuki    # 本気層・契約書送付の文面を生成
  /brief 2026-05-02-sato-miki          # 明日の面談準備
  /classify 2026-04-28-suzuki-ken      # 分類未実行
```

## 制約
- `candidates/_example/` は除外する
- 期日の相対計算は Bash の `date` コマンドで今日の日付を取得して行う
- 欠損情報（期日が未記載など）は `[期日不明]` として別セクションに
