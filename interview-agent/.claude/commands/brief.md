# /brief — 候補者ブリーフ生成

面談開始 5 分前に、候補者の SNS と事前情報から **面談に持ち込める 1 枚ブリーフ** を生成します。

## 引数
- `$1` : 候補者 slug（`candidates/<slug>/` のディレクトリ名）

## 手順（Claude への指示）

### Step 1: プロフィール読み込み
1. `candidates/$1/profile.md` を Read する
2. ファイルがなければ、ユーザーに「`candidates/$1/profile.md` を先に作成してください（氏名・年齢・SNS URL・紹介元など）」と報告して終了

### Step 2: SNS 分析（サブエージェント委譲）
`profile.md` に記載されている SNS URL ごとに、`sns-analyzer` サブエージェントを **並列で** 呼び出す。

サブエージェントには以下を渡す:
- SNS URL
- 候補者の氏名（あれば）
- 抽出してほしい項目（persona / 活動量 / 志向性 / 面談で触れるべき話題 / 注意点）

### Step 3: ブリーフ合成
サブエージェントの結果を統合し、`templates/brief.md` のフォーマットに沿って **A4 1 枚**（600–900 字）でまとめる。

必須セクション:
1. **基本情報**（氏名／年齢推定／所在地／紹介元）
2. **未成年警告**（年齢が 18 歳未満の可能性がある場合のみ、冒頭に赤字相当の注意書き）
3. **ペルソナ 3 行要約**
4. **動機の仮説**（なぜモデル／アクターを目指すのか、SNS からの推定）
5. **面談で触れるべき 3 つのフック**（共通の話題、褒めポイント、共感点）
6. **想定される不安・反論**（価格、時間、家族、業界不安 等）
7. **推奨トーク運び**（オープニング 1 行案、クロージング方向性）
8. **レッドフラグ**（冷やかし兆候、炎上歴、矛盾等があれば）
9. **🕐 面談直前 5 分チェックリスト**（必ず末尾に配置）:
   - [ ] `raw-notes.md`（チェックリスト形式）を開いた
   - [ ] 録音許諾の確認タイミングを決めた
   - [ ] ★ マーク必須項目を把握した（動機 / 目標 / 予算 / 決裁）
   - [ ] クロージングで何に落とし込むか決めた（契約 / 体験 / 資料送付）
   - [ ] 未成年対応がある場合、保護者同意の話し方を確認した

### Step 4: 保存と報告
- 結果を `candidates/$1/brief.md` に Write
- ユーザーには **ファイルパスと 3 行サマリー** だけを報告（ブリーフ全文は貼らない）

## 制約
- 推測には必ず「（推定）」を付与
- 容姿の数値評価（点数など）は禁止
- 1 枚に収まらない場合は「詳細は省略、`brief-full.md` 参照」とする（ただし v1 では 1 枚厳守）
- SNS 取得に失敗したら、その SNS は「取得不可」と明記して続行

## 出力例（ユーザーへの最終報告）
```
✅ ブリーフ生成完了: candidates/2026-04-21-tanaka-yuki/brief.md

3 行サマリー:
- 21 歳女性（推定）、大阪在住、TikTok フォロワー 8k。
- ダンスと K-POP が強い関心。芸能への憧れは 1 年前から明確化。
- 懸念: 家族の反対の可能性、価格センシティブ。家族同席オファーが刺さる可能性。

推奨の次アクション:
  /rehearse 2026-04-21-tanaka-yuki   # 面談前に 10 分で練習
  面談後 → /minutes → /classify → /followup
```
