# /minutes — 面談議事録生成

面談直後に、生メモ／文字起こしを **構造化された議事録** に変換します。

## 引数
- `$1` : 候補者 slug

## 入力ファイル
以下を探して、存在するものは **全て併用** する:

1. `candidates/$1/transcript.md` — Whisper 文字起こし（タイムスタンプ付き、最も網羅的）
2. `candidates/$1/raw-notes.md` — 面談中の生メモ（チェックリスト形式、担当者の解釈入り）
3. `candidates/$1/raw-notes.txt` — 生メモのテキスト形式

### 入力ソースによる扱いの違い
- **transcript.md のみ**: 逐語記録として信頼度高。発言ログには **タイムスタンプ付きで多めに**（5–10 件）引用する。
- **raw-notes.md のみ**: 担当者の一次解釈ベース。引用は raw-notes 内の記載から。
- **両方存在（推奨）**: raw-notes を「担当者の解釈」、transcript を「verbatim 裏付け」として扱う。齟齬があれば **transcript を正** として採用し、齟齬があった事実を議事録末尾の所感に記載。

いずれも無ければ、ユーザーに以下を案内して終了:
```
入力ファイルがありません。以下のいずれかを用意してください:
  1. raw-notes.md を手書き（candidates/$1/raw-notes.md）
  2. 音声ファイルを置いて /transcribe $1 を実行
```

## 手順

### Step 1: 入力読み込み
上記の 3 ファイルを（存在するだけ）すべて Read する。`brief.md` があれば合わせて読み、面談前仮説と照合する。

### Step 2: 構造化（`templates/minutes.md` 準拠）
以下のセクションに情報を配置する。発言は可能な限り **verbatim（逐語）で引用** し、候補者の言葉遣いを残す。

1. **ヘッダー**（日時、場所、面談時間、担当者）
2. **基本情報アップデート**（brief からの差分のみ）
3. **動機・目標**
   - なぜモデル／アクターを目指すか
   - いつまでに何を達成したいか
4. **現在地の自己認識**（経験、強み、弱みの自己評価）
5. **障壁・不安**（価格／時間／家族／業界不信 等、全て列挙）
6. **予算感・決定プロセス**
   - 価格感度（高／中／低）
   - 決裁者（本人のみ／保護者／配偶者）
   - 資金源
7. **レッスン希望条件**（曜日・時間帯・頻度・対面/オンライン）
8. **刺さった訴求 / 刺さらなかった訴求**
9. **次アクション TODO**（期日付きで最大 5 件）
10. **発言ログ**（印象的な逐語引用を 3–7 個、コンテキスト付き。transcript.md があれば **タイムスタンプ付き**で引用する: 例 `[00:12:34]`）

### Step 3: 未確認項目を明示
面談で確認できなかった重要項目は **`[未確認]` タグ付き** で列挙する。
（例: `[未確認] 予算感`、`[未確認] 保護者同席可否`）

### Step 4: 保存と報告
- `candidates/$1/minutes.md` に Write
- ユーザーには **5 行サマリー** だけ報告:
  - 候補者の動機の核（1 行）
  - 最大の障壁（1 行）
  - 最も刺さった訴求（1 行）
  - 次アクション最優先 1 件（1 行）
  - 未確認の重要項目（1 行）

## 制約
- **生メモにない情報を創作しない**。推測は「（推定）」と明示
- 発言の引用は、面談者の許諾範囲内であることを前提とする
- 容姿評価・差別的表現は議事録からも削除する
- 議事録の後には必ず `/classify $1` を実行することをユーザーに促す

## 出力例
```
✅ 議事録生成: candidates/2026-04-21-tanaka-yuki/minutes.md

サマリー:
- 動機: 3 年後までに CM 1 本。現職（アパレル販売）のままではキャリア不安。
- 最大障壁: 月額 3 万超えると厳しい。家族未相談。
- 刺さった: 「卒業生の大阪ローカルタレント事例」
- 次アクション: 家族向け資料 PDF を本日中に送付
- [未確認] 保護者同席 OK か、貯金の具体額

次は /classify 2026-04-21-tanaka-yuki を実行してください。
```
